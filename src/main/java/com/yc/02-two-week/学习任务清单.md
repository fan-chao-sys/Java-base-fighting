## Day 1：集合体系结构、List / Set / Map 区别

### 📖 基础知识点
- [ ] Java 集合框架体系：`Collection`（List + Set + Queue）和 `Map` 两大分支
- [ ] List：有序、可重复、按索引访问（`ArrayList`、`LinkedList`、`Vector`）
- [ ] Set：无序（除 LinkedHashSet）、不可重复（`HashSet`、`LinkedHashSet`、`TreeSet`）
- [ ] Map：键值对、key 不可重复（`HashMap`、`LinkedHashMap`、`TreeMap`、`Hashtable`）
- [ ] `Comparable`（内部排序，`compareTo`）vs `Comparator`（外部排序，`compare`），排序优先级：Comparator > Comparable

### ❓ 常见面试题
**Q1: `Comparable` 和 `Comparator` 的区别？**
> 答：`Comparable` 是内部排序接口，实现类重写 `compareTo`，类自身具备比较能力（如 `String`）；`Comparator` 是外部排序接口，独立于类，可定义多种排序策略（如 `Comparator.comparing(User::getAge)`）。Comparator 优先级更高。

**Q2: `Vector`、`ArrayList`、`LinkedList` 三者区别？**
> 答：`Vector` 线程安全（synchronized），扩容翻倍，已过时；`ArrayList` 非线程安全，数组实现，查询 O(1)，扩容 1.5 倍；`LinkedList` 非线程安全，双向链表实现，插入删除 O(1)（已定位到节点前提下）。

## Day 2：精看 ArrayList，底层数组、扩容、随机访问

### 📖 基础知识点
- [ ] `ArrayList` 底层：`Object[] elementData`，默认容量 10（JDK 7+ 采用懒加载，首次 add 才分配）
- [ ] 扩容机制：容量不够时 `grow()`，新容量 = 旧容量 + 旧容量 >> 1（即 1.5 倍），`Arrays.copyOf()` 复制元素
- [ ] 随机访问 O(1)：基于数组下标直接定位
- [ ] 插入/删除中间元素 O(n)：需要 `System.arraycopy()` 移动后续元素
- [ ] `modCount`：记录结构修改次数，迭代器检测到 `modCount` 变化则抛出 `ConcurrentModificationException`（fail-fast）

### ❓ 常见面试题
**Q1: `ArrayList` 扩容机制是怎样的？**
> 答：ensureCapacityInternal → 如果元素数组是空（首次添加），取 max(10, minCapacity) 分配；非首次扩容 → grow(minCapacity) → newCapacity = oldCapacity + (oldCapacity >> 1)，即 1.5 倍 → 如果还不够，直接用 minCapacity → 如果超过 MAX_ARRAY_SIZE，取 Integer.MAX_VALUE → 最后 `Arrays.copyOf()` 复制。

**Q2: `ArrayList` 的 modCount 是干什么的？**
> 答：记录结构修改次数（add/remove 等），迭代器在遍历时比较 `expectedModCount` 和 `modCount`，不一致则抛 `ConcurrentModificationException`。这是 fail-fast 机制，用于检测"迭代期间集合被其他线程或本线程修改"。

## Day 3：精看 LinkedList，链表结构、插入删除、遍历成本

### 📖 基础知识点
- [ ] `LinkedList` 底层：双向链表，`Node` 包含 prev / item / next 三个指针
- [ ] 插入删除：头尾 O(1)，中间需先遍历找到位置（O(n)），所以"插入删除快"前提是已定位
- [ ] 实现了 `Deque` 接口，可作栈（push/pop）或队列（offer/poll）使用
- [ ] 随机访问 O(n)：`get(index)` 需要从头/尾遍历到目标位置
- [ ] `ArrayList` vs `LinkedList` 遍历性能：ArrayList 用 for 最快（内存连续，CPU 缓存友好），LinkedList 用迭代器/增强 for（避免每次 get 都 O(n)）

### ❓ 常见面试题
**Q1: `ArrayList` 和 `LinkedList` 怎么选？**
> 答：查多写少 → ArrayList（随机访问 O(1)，内存连续）；频繁在头尾增删 → LinkedList（O(1)）；频繁在中间增删 → 差距不大（都需定位）；遍历 → ArrayList 用普通 for，LinkedList 用迭代器。大部分场景 ArrayList 是更好的选择。

## Day 4：精看 HashMap（上），数组 + 链表 + 红黑树、put 流程、扩容

### 📖 基础知识点
- [ ] `HashMap` 底层：`Node<K,V>[] table`，数组 + 链表 + 红黑树（JDK 8+）
- [ ] put 流程：① 计算 hash（`(h = key.hashCode()) ^ (h >>> 16)`，高低位混合）→ ② 定位桶位置 `tab[i = (n-1) & hash]` → ③ 桶为空直接放 → ④ 桶有元素，判断 key 是否相等，相等则覆盖 → ⑤ 不等则判断是链表还是红黑树，链表尾插，树则插入树 → ⑥ 链表长度 ≥ 8 且数组长度 ≥ 64 则树化 → ⑦ `++size > threshold` 则扩容
- [ ] 哈希冲突解决：JDK 7 头插法 + 链表，JDK 8 改为尾插法 + 链表/红黑树（避免死循环 + 优化长链查询 O(n)→O(logn)）
- [ ] 扩容：容量翻倍，每个桶的元素重新计算位置（原位置 或 原位置+旧容量）

### ❓ 常见面试题
**Q1: `HashMap` 的 put 流程每一步干了什么？**
> 答：验证 table 是否初始化 → 桶为空直接放 → 桶首节点 key 相同则替换 → 判断红黑树/链表 → 遍历链表（计数）→ key 相同替换，不同尾插 → 链表长度 ≥ 8 且 table.length ≥ 64 转红黑树 → size > threshold 扩容。

**Q2: `HashMap` 为什么容量是 2 的幂？**
> 答：`tab[i = (n-1) & hash]` 位运算取模，当 n=2^k 时，`(n-1) & hash` 等价于 `hash % n`，但位运算比取模快得多。同时保证索引均匀分布，减少哈希冲突。

## Day 5：精看 HashMap（下），JDK 1.7 vs 1.8 差异、树化条件

### 📖 基础知识点
- [ ] JDK 7 vs JDK 8 差异：
  - 结构：7 只有数组+链表；8 引入红黑树
  - 插入：7 头插法（resize 时可能形成环形链表导致死循环）；8 尾插法（安全）
  - hash 计算：7 更复杂（多次扰动）；8 简化为 `(h ^ h>>>16)`
  - 扩容条件：8 还要判断链表长度 ≥ 8 且数组长度 ≥ 64 才树化
- [ ] 树化条件：链表长度 ≥ 8（因为节点频率服从泊松分布，概率 < 千万分之一）且 table.length ≥ 64（数组太小时优先扩容）
- [ ] 退化条件：树节点 ≤ 6 时退化为链表
- [ ] `loadFactor` 默认 0.75：时间与空间的折中，过高则查找变慢，过低则空间浪费

### ❓ 常见面试题
**Q1: `HashMap` 1.7 和 1.8 的关键区别？**
> 答：① 1.7 数组+链表，1.8 数组+链表+红黑树（长链 O(n) → O(logn)）；② 1.7 头插法 resize 可能死循环（环形链表），1.8 尾插法无此问题；③ 1.8 优化了 hash 算法和扩容机制。

**Q2: 什么时候链表会树化？什么时候退化为链表？**
> 答：树化：链表长度 ≥ 8 且数组长度 ≥ 64（否则优先扩容）。退化：树节点 ≤ 6 时转为链表。8 和 6 中间有 7 的缓冲，避免频繁转换。

## Day 6：ConcurrentHashMap、fail-fast、迭代器机制

### 📖 基础知识点
- [ ] `Hashtable`：全表锁 `synchronized`，并发性能差
- [ ] `ConcurrentHashMap` JDK 7：分段锁 Segment（默认 16 段），不同 Segment 可并发操作
- [ ] `ConcurrentHashMap` JDK 8：CAS + synchronized（锁桶首节点），更细粒度；结构也与 HashMap 统一（数组+链表+红黑树）
- [ ] `HashSet` 底层就是 `HashMap`，元素作为 key，value 是固定的 `PRESENT` 对象
- [ ] 迭代器的 fail-fast：`ConcurrentHashMap` 的迭代器是弱一致性（fail-safe），不抛 `ConcurrentModificationException`，遍历时允许并发修改
- [ ] `Collections.synchronizedMap()` 包装普通 Map 为同步 Map

### ❓ 常见面试题
**Q1: `ConcurrentHashMap` 为什么比 `Hashtable` 更好？**
> 答：`Hashtable` 用全表锁，同一时刻只有一个线程能操作；JDK 7 CHM 用分段锁（默认 16 段），不同段可并发，并发度 16；JDK 8 CHM 用 CAS + synchronized 锁桶首节点，粒度更细，且读操作完全无锁（volatile 保证可见性）。

**Q2: `HashSet` 为什么元素不能重复？**
> 答：`HashSet` 底层基于 `HashMap`，add 时调 `map.put(e, PRESENT)`，依靠 HashMap 的 key 唯一性保证元素不重复。如果两个元素的 `hashCode` 相等且 `equals` 返回 true，后者会覆盖前者。
