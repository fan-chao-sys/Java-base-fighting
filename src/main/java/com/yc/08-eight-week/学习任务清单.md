## Day 1：数组、链表、栈、队列、哈希经典题

### 📖 基础知识点
- [ ] 数组：连续内存，O(1) 随机访问，插入删除 O(n)；双指针技巧（左右指针/快慢指针）
- [ ] 链表：非连续内存，O(n) 查找，O(1) 插入删除（已定位）；重点掌握哨兵节点简化边界
- [ ] 栈：LIFO，`Deque` 替代 `Stack`（Stack 是遗留类，性能差）；典型场景：括号匹配、表达式求值、DFS 非递归
- [ ] 队列：FIFO，`LinkedList` / `ArrayDeque` 实现；典型场景：BFS、滑动窗口
- [ ] 哈希：O(1) 查找；典型场景：两数之和、字符统计、去重

### ❓ 常见面试题
**Q1: 反转链表（迭代和递归两种）**
> 迭代：pre=null, cur=head, 循环保存 next, cur.next=pre, pre/cur 后移；递归：`reverseList(head.next)` 然后 `head.next.next=head; head.next=null`。

**Q2: 环形链表检测**
> 快慢指针：slow 走 1 步，fast 走 2 步，相遇则有环。找环入口：相遇后一个指针回 head，两个都走 1 步，再次相遇即为入口。

## Day 2：二叉树、DFS、BFS、递归套路

### 📖 基础知识点
- [ ] 二叉树遍历模板：前序（根→左→右）、中序（左→根→右）、后序（左→右→根），递归和迭代写法都要会
- [ ] BFS：层次遍历，用队列，逐层处理（记录每层 size）
- [ ] DFS：递归（天然的栈调用）或非递归（手动用 Stack）
- [ ] 递归套路：明确函数定义（做什么、返回什么）→ 确定终止条件 → 把问题分解为子问题 → 合并结果
- [ ] 常见变种：二叉树最大/最小深度、对称二叉树、翻转二叉树、二叉树的最近公共祖先

### ❓ 常见面试题
**Q1: 二叉树层次遍历怎么实现？**
> 答：使用队列 BFS。根节点入队 → 循环弹出，记录当前层 size → 逐个处理 size 个节点，把子节点入队。时间复杂度 O(n)，空间复杂度 O(n)。

**Q2: 二叉树的最近公共祖先（LCA）**
> 答：递归：① 当前节点为空或等于 p 或 q，返回当前节点；② 递归左右子树；③ 左右都非空则当前节点是 LCA；④ 否则返回非空的那一边。

## Day 3：双指针、滑动窗口、二分、堆

### 📖 基础知识点
- [ ] 双指针：左右指针（有序数组两端向内收缩，如两数之和 II、盛水容器）；快慢指针（链表判环、中间节点）
- [ ] 滑动窗口：维护 [left, right) 窗口，right 扩大找可行解，left 收缩求最优解。模板：right 滑动更新状态 → while 满足条件 → left 滑动更新状态
- [ ] 二分查找：前提有序；注意边界（左闭右闭 `[l,r]` vs 左闭右开 `[l,r)`），避免死循环用 `mid = l + (r-l)/2`
- [ ] 堆（PriorityQueue）：默认小顶堆，可传 Comparator 变行为；Top K 场景首选

### ❓ 常见面试题
**Q1: 最长无重复子串（滑动窗口）**
> 答：HashMap 存字符最后出现位置 + 滑动窗口。right 遍历，有重复则 left 跳到 max(left, map.get(c)+1)，更新 maxLen。

**Q2: Top K 问题用什么数据结构？**
> 答：堆（PriorityQueue）。求最大的 K 个用小顶堆（堆顶是 K 个中最小的，新来的比堆顶大就替换），求最小的 K 个用大顶堆。时间复杂度 O(nlogk)。

## Day 4：动态规划入门，重点理解状态定义与转移

### 📖 基础知识点
- [ ] DP 三要素：状态定义（dp[i] 代表什么）→ 状态转移方程（dp[i] 和之前状态的关系）→ 初始值和边界条件
- [ ] 通过经典题理解：爬楼梯（线性 DP）、打家劫舍（状态机 DP）、最长递增子序列（一维 DP）、买卖股票（多维状态 DP）
- [ ] DP 优化：空间压缩（滚动数组，只保留 dp[i-1] 和 dp[i-2]）
- [ ] 什么时候用 DP：求最值/计数/是否存在，且问题可分解为重叠子问题

### ❓ 常见面试题
**Q1: 买卖股票的最佳时机（单次交易）**
> 答：记录到当前为止的最低价格 minPrice，每天计算 `max(前一日利润, 当日价格-minPrice)`。一次遍历 O(n)。

**Q2: 爬楼梯问题**
> 答：dp[i] = dp[i-1] + dp[i-2]（到第 i 阶的方法数=从 i-1 跨 1 步 + 从 i-2 跨 2 步）。dp[0]=1, dp[1]=1。本质是斐波那契数列，可压缩为两个变量滚动。

### 💻 代码实战

- [ ] 2. 买卖股票的最佳时机 ——LeetCode 121
- [ ] 3. 最长递增子序列 ——LeetCode 300（O(n²) 先写，再了解 O(nlogn) 解法）

## Day 5：Linux 常用命令与线上排查流程

### 📖 基础知识点
- [ ] 日志查看：`tail -f -n 200 app.log`（实时滚动）、`grep "ERROR" app.log | tail -20`（过滤最新异常）、`less app.log`（大文件浏览，`?pattern` 向上搜索，`/pattern` 向下搜索）
- [ ] 进程排查：`top` / `htop`（CPU/内存排序：按 P/M）、`ps -ef | grep java`（进程列表）、`lsof -p <pid>`（打开的文件/端口）
- [ ] 端口：`netstat -tlnp`（监听端口）、`lsof -i :8080`（端口被谁占用）、`ss -tlnp`（更快的 netstat 替代品）
- [ ] Java 工具链：`jps -lvm`（查看 Java 进程+参数）、`jstack <pid>`（线程栈，排查死锁/线程阻塞）、`jmap -heap <pid>`（堆概况）、`jmap -dump:live,format=b,file=heap.bin <pid>`（dump 堆快照）、`jstat -gcutil <pid> 1000 10`（每秒打印 GC 情况）
- [ ] 性能监控：`vmstat 1`（系统整体）、`iostat -x 1`（磁盘 IO）、`free -h`（内存使用）

### ❓ 常见面试题
**Q1: CPU 飙高怎么排查？**
> 答：① `top` 找到 CPU 最高的 Java 进程 PID → ② `top -Hp <pid>` 找到 CPU 最高的线程 TID → ③ `printf '%x\n' <tid>` 转 16 进制 → ④ `jstack <pid> | grep -A 30 <nid>` 定位到线程栈 → ⑤ 看是 GC 线程（考虑内存问题）、业务线程（查代码热点）、死循环 → ⑥ `jstat -gcutil <pid>` 确认是否 GC 引起。

**Q2: 内存溢出怎么排查？**
> 答：① `jmap -dump:format=b,file=heap.bin <pid>` 导出 dump → ② 用 MAT 分析 dump（看直方图、大对象、Leak Suspects）→ ③ 定位到具体对象和 GC Root 路径 → ④ 如果是堆溢出：优化代码减少对象创建或调大堆；如果是元空间溢出：检查是否大量动态生成类。
