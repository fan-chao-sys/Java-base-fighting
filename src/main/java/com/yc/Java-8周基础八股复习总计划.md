# Java 基础与八股复习计划

## 1. 这份计划怎么用
- 每一周都围绕一个主题，做到“能理解、能输出、能手写、能结合项目讲”。
- 建议工作日每天投入 2 小时，周末每天投入 4 到 6 小时。
- 每周都要产出自己的笔记，不要只看不说。
- 每个知识点都按同一套模板复习：
```text
1. 它是什么
2. 它解决什么问题
3. 底层怎么实现
4. 优缺点和适用场景
5. 项目里怎么用
6. 常见坑和排查思路
```

## 2. 总体复习策略

## 4. 八股复习的答题方法

很多人不是不会，而是讲不顺。建议统一用下面这个答题结构：

```text
第一句：先下定义
第二句：说明它解决的问题
第三句：讲核心实现原理
第四句：讲优缺点或边界
第五句：结合项目举例
```

例如回答“`HashMap` 为什么线程不安全”：

```text
HashMap 是一个非线程安全的哈希表实现，主要用于单线程场景。
它的问题本质是多个线程同时修改数组和链表/红黑树结构时，没有任何同步保护。
在并发 put、resize 时，可能出现数据覆盖、丢失、读取不一致等问题，JDK 1.7 里极端情况下还可能形成链表环。
所以并发场景一般会用 ConcurrentHashMap，而不是直接用 HashMap。
我们项目里缓存本地配置时用过 HashMap，但只在启动阶段初始化，运行期不并发修改，因此没有线程安全问题。
```
