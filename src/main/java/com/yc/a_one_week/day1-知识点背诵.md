# 一、Java 基本数据类型（Primitive Type）核心总览

## 1\.1 核心定义

Java 共 **8种基本数据类型**，是JVM内置、非对象、无继承关系、直接存数值的基础类型。

**四大分类**：整数型、浮点型、字符型、布尔型

**核心底层特征**：

- 直接存储**数值本身**，不存引用地址

- 内存固定、平台无关（跨语言一次编译到处运行）

- 存储在**栈内存**，访问速度极快

- 无方法、无属性，不能调用任何API

## 1\.2 8种基本类型最全参数表（必背）

|---|---|---|---|---|---|---|---|

## 1\.3 基本类型底层存储原理（必懂）

### 1\.3\.1 内存存储机制

- **基本类型变量存栈内存**：<u>变量名</u> 和 <u>数值</u> 直接在 <u>栈帧中</u>，<u>无堆内存占用</u>       ⭐

- **读写效率极高**：无需指针寻址、无需GC回收

- **线程私有**：栈内存线程私有，无线程安全竞争问题

### 1\.3\.2 值传递底层

基本类型作为方法参数：**传递数值副本**

方法内部修改参数，**不会影响原变量**，因为是两个独立的栈内存数值。

### 1\.3\.3 类型转换底层规则（必考）

**1）隐式自动转换（编译自动完成）**

规则：**小范围 → 大范围，安全无精度丢失**

转换顺序：`byte → short → int → long → float → double`

特殊：`char → int`（char无符号，可直接转整型）⭐

**2）显式强制转换（手动强转）**

规则：**大范围 → 小范围，存在截断、精度丢失**

示例：`int a = \(int\) 3\.9; // 结果为3，小数直接截断`

**核心结论**：浮点型优先级高于整型，任何整型转浮点自动转换，浮点转整型必须强转。

### 1\.3\.4 浮点精度丢失底层原因（高频面试）⭐

`float / double` 遵循 **IEEE754 浮点数标准**，采用 **<u>二进制</u>** 科学计数法存储。

<u>十进制 </u>小数（0\.1、0\.2）无法被 <u>二进制</u> 精准表示，属于无限循环小数，存储时截断，导致精度丢失。

**业务规范**：金融、金额计算绝对禁止使用float/double，必须使用 `BigDecimal`。

# 二、包装类（Wrapper Class）底层核心

## 2\.1 包装类存在的意义（底层设计思想）

基本类型是**裸数据**，不满足Java面向对象特性，存在两大缺陷：

1. <u>无法存入**集合、泛型**（集合只能存对象）</u>

2. <u>没有任何工具方法</u>，<u>无法完成字符串转换、数值判断</u>等操作

**包装类作用**：将基本类型**封装为对象**，适配面向对象体系，提供大量工具方法。

所有包装类均在 `java\.lang` 包，默认无需导包。

## 2\.2 装箱与拆箱（Java5 新特性）

### 2\.2\.1 概念定义

- **装箱**：基本类型 → 包装对象

- **拆箱**：包装对象 → 基本类型

### 2\.2\.2 底层代码本质（编译器语法糖）

自动装箱/拆箱是 **编译期语法糖**，运行期无任何特殊机制，编译器自动补全代码。

**自动装箱底层**：默认调用 `Integer\.valueOf\(\)`

**自动拆箱底层**：默认调用 `intValue\(\)`

### 2\.2\.3 代码对照

```Plain Text
// 自动装箱
Integer a = 10; 
// 编译后底层：Integer a = Integer.valueOf(10);

// 自动拆箱
int b = a;
// 编译后底层：int b = a.intValue();
```

## 2\.3 <u>包装类缓存池</u>机制（面试重中之重）

### 2\.3\.1 缓存池设计目的

高频小数值使用频次极高，为了**减少对象创建、节省堆内存、提升效率**，Java 对部分包装类提供缓存池。

### 2\.3\.2 所有缓存范围（必背）

- `Byte、Short、Integer、Long`：缓存 **\-128 \~ 127**  ⭐

- `Character`：缓存 **0 \~ 127**（ASCII可见字符）        ⭐

- `Boolean`：缓存静态常量 TRUE / FALSE                  ⭐

- **无缓存**：Float、Double（浮点数值无限，无法缓存） ⭐

### 2\.3\.3 缓存底层原理（以Integer为例）

Integer 内部存在私有静态内部类 `IntegerCache`。

类加载时，一次性初始化 \-128\~127 所有Integer对象，存入数组缓存。

调用 `valueOf\(\)` 时：

- 数值在缓存区间：**直接返回缓存对象**

- 数值超出缓存区间：**new 新对象**

### 2\.3\.4 经典面试代码

```Plain Text
Integer a = 127;
Integer b = 127;
System.out.println(a == b); // true （复用缓存）

Integer c = 128;
Integer d = 128;
System.out.println(c == d); // false（超出缓存，两个新对象）
```

## 2\.4 包装类核心特性

### 2\.4\.1 不可变性

<u>所有包装类都是 **不可变类**</u>（<u>被final修饰，底层数值变量final</u>）。

一旦对象创建，数值无法修改，所谓“修改值”本质是**新建包装对象**。

### 2\.4\.2 默认值区别（超级易错）

- 基本类型：有具体默认值（0、0\.0、false）

- 包装类型：默认值 **null**（空对象，不指向任何堆内存）

### 2\.4\.3 核心工具方法（开发常用）

- `xxx\.parseXxx\(String\)`：字符串转基本类型（重点，抛数字格式异常）

- `Xxx\.valueOf\(\)`：优先创建包装对象，走缓存机制

- `xxxValue\(\)`：包装类转基本类型（手动拆箱）

## 2\.5 包装类 hashCode \&amp; equals 底层核心（面试高频）

### 2\.5\.1 核心底层规则

所有数值型包装类（Integer、Long、Double等）、Character、Boolean，均**重写了 hashCode\(\) 和 equals\(\) 方法**，核心逻辑只比较**真实存储的数值**，不比较对象地址。

### 2\.5\.2 重写底层原理

- **hashCode\(\) 重写**：直接返回包装对象存储的基本数值（如Integer直接返回int值，Long返回long值），保证**相同数值的包装对象，哈希码完全一致**。

- **equals\(\) 重写**：先判断对象类型是否一致，再对比内部存储的数值，**数值相同即返回true**，彻底忽略对象地址差异。

### 2\.5\.3 代码实战对比（彻底解决==坑）

```java
// 超出缓存范围，两个不同对象地址
Integer a = 128;
Integer b = 128;

System.out.println(a == b);        // false：比较对象地址
System.out.println(a.equals(b));   // true：比较底层数值（重写后逻辑）
System.out.println(a.hashCode() == b.hashCode()); // true：数值相同哈希码相同
```

### 2\.5\.4 终极结论（必背）

1. 包装类 **== 比地址**，受缓存池影响；

2. 包装类 **equals/hashCode 比数值**，不受缓存池影响；

3. 业务开发中，包装类数值比较**必须使用equals\(\)**，禁止使用==。

## 2\.6 Java 值传递 \&amp; 引用传递 底层吃透（重难点）

### 2\.6\.1 Java 唯一传参机制：纯值传递

**核心定论（面试必考）**：Java 中只有**值传递**，不存在引用传递！所有方法参数传递，本质都是**传递副本**。

### 2\.6\.2 两类参数传递底层区别

#### 1\. 基本类型参数传递

传递：**原始数值的副本**

底层：栈中复制一份全新数值，方法内修改副本，**完全不影响原变量**。

```java
public static void change(int num){
    num = 100; // 修改的是副本数值
}
public static void main(String[] args) {
    int a = 10;
    change(a);
    System.out.println(a); // 输出10，原变量无变化
}
```

#### 2\. 包装类/引用类型参数传递

传递：**对象引用地址的副本**（极易混淆点）

底层：栈中复制一份对象引用地址，原变量和参数**指向同一个堆对象**；但引用地址本身是副本，无法修改原引用的指向。

### 2\.6\.3 包装类专属传递坑点

包装类是**不可变类**，一旦赋值无法修改内部数值。方法中修改包装类参数，本质是**让副本引用指向新对象**，原引用指向的旧对象完全不变。

```java
public static void change(Integer num){
    num = 100; // 副本引用指向新对象100
}
public static void main(String[] args) {
    Integer a = 10;
    change(a);
    System.out.println(a); // 输出10，原对象无变化
}
```

### 2\.6\.4 核心总结（秒懂区别）

- **基本类型**：传【数值副本】→ 修改副本 → 原值不变

- **包装/引用类型**：传【地址副本】→ 修改副本指向 → 原引用不变；修改副本指向的对象内容 → 原对象内容改变

- **统一本质**：所有参数传递，都是**拷贝副本**，无任何例外

# 四、底层高频考点\&amp;易错坑点（必背避坑）

## 4\.1 空指针异常坑

包装类为 **null** 时，自动拆箱直接报`NullPointerException`。

原因：null对象无法调用 `xxxValue\(\)` 拆箱方法。

## 4\.2 == 与 equals 坑

- `==`：比较**对象地址**（缓存范围内true，范围外false）

- `equals\(\)`：比较**数值内容**（推荐业务使用）

## 4\.3 char 类型特殊点

- char 是**无符号类型**，取值0\~65535

- char 可以直接参与数学运算，使用Unicode编码值计算

## 4\.4 boolean 虚拟机底层

JVM 没有专门的boolean指令，底层用int替代：

- 1 = true

- 0 = false

boolean数组底层采用byte数组实现，节省空间。

## 4\.5 自动装箱复用原则

开发中创建包装对象，**优先使用 valueOf\(\)**，可以复用缓存对象，减少内存开销。

# 五、终极背诵总结（面试秒答）

1. 8大基本类型分四类：4整、2浮、1字符、1布尔

2. 基本类型栈存、无对象、速度快；包装类堆存、是对象、可null

3. 装箱拆箱是编译语法糖，底层valueOf/xxxValue

4. 包装类不可变、默认null，比较值必须用equals，hashCode与equals均基于数值重写

5. Java只有值传递，基本类型传数值副本，包装类传引用地址副本，均不影响原变量指向

6. 浮点精度丢失源于IEEE754二进制存储缺陷，金额用BigDecimal
