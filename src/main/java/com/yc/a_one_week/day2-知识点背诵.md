# Day2 Java面向对象核心知识点背诵笔记（底层吃透版）

---

## 一、面向对象三大特性（封装、继承、多态）

### 1. 封装（Encapsulation）
#### 核心定义
隐藏对象的内部状态和实现细节，仅对外提供受控的访问接口，实现高内聚、低耦合。

#### 底层原理
- 访问控制修饰符实现权限隔离：`private`（类内可见）→ `default`（包内可见）→ `protected`（包+子类可见）→ `public`（全局可见）。
- 通过`private`修饰成员变量，提供`getter/setter`方法暴露受控访问接口，在方法中可增加校验逻辑。

#### 核心特性
- 安全性：防止外部随意修改内部状态，避免非法值写入。
- 可维护性：修改内部实现时，只要接口不变，外部调用不受影响。

---

### 2. 继承（Inheritance）
#### 核心定义
子类（派生类）复用父类（基类）的属性和方法，实现代码复用与扩展，体现“is-a”关系。

#### 底层原理
- Java仅支持**单继承**：一个类只能直接继承一个父类，避免多继承的菱形继承问题。⭐
- 子类无法继承父类的`private`成员，但可通过父类的`public/protected`方法间接访问。⭐
- 构造器调用顺序：父类构造器先执行，再执行子类构造器（`super()`默认调用父类无参构造器）。

#### 核心特性
- 方法重写（Override）的前提：子类继承父类非私有方法后，可重写其实现。
- 向上转型：父类引用指向子类对象，是多态的基础。

---

### 3. 多态（Polymorphism）
#### 核心定义
同一操作作用于不同对象时，产生不同的执行结果，核心体现为**方法的动态绑定**。

#### 底层原理
- 实现前提：父类引用指向子类对象 + 子类重写父类方法。
- 绑定机制：
  - 编译期绑定（静态绑定）：方法重载、静态方法、final方法，编译时确定调用哪个方法。
  - 运行期绑定（动态绑定）：方法重写，运行时根据对象的实际类型确定调用的方法（“编译看左边，运行看右边”）。
- JVM底层：通过方法表（Method Table）实现动态绑定，每个类维护一个方法表，存储方法的实际地址，运行时根据对象类型查表调用。

#### 核心特性
- 提高代码的可扩展性：新增子类无需修改原有代码，仅需新增实现即可。
- 降低耦合度：父类定义接口规范，子类负责具体实现。

---

## 二、方法重载（Overload）与重写（Override）

### 1. 方法重载（Overload）
#### 核心定义
同一类中，方法名相同，但参数列表（参数个数、类型、顺序）不同的多个方法，编译期绑定。

#### 底层原理
- 编译器通过方法签名（方法名+参数列表）区分不同重载方法，返回值和修饰符不参与重载判断。
- 本质是编译期的静态绑定，JVM根据调用时传入的参数类型确定调用哪个方法。

#### 核心规则
- 必须满足：方法名相同，参数列表不同。
- 不参与判断：返回值类型、访问修饰符、异常类型。

---

### 2. 方法重写（Override）
#### 核心定义
子类中定义与父类中同名、同参数列表、同返回值类型的方法，覆盖父类的实现，运行期绑定。

#### 底层原理
- 子类重写方法时，方法表中对应父类方法的地址会被子类方法的地址替换，运行时直接调用子类方法。
- 遵循“里氏替换原则”：子类对象可替换父类对象使用，不破坏原有逻辑。

#### 核心规则（必背）
- 方法签名必须与父类完全一致（方法名+参数列表）。
- 返回值类型：协变返回（子类返回值类型可以是父类返回值类型的子类）。
- 访问权限：子类方法的访问权限不能比父类更严格（如父类protected，子类不能为private）。
- 异常：子类方法不能抛出比父类方法更宽泛的异常（可抛出更窄的异常或不抛出异常）。
- 不能重写的方法：`final`方法、`static`方法、`private`方法（子类定义同名方法不属于重写，是隐藏）。

---

## 三、final与static关键字

### 1. final关键字
#### 核心作用
用于修饰类、方法、变量，表达“不可修改”的语义。

#### 底层原理与特性
- **修饰类**：该类不可被继承，所有方法默认被隐式final修饰（如String类）。
- **修饰方法**：该方法不可被子类重写，编译期绑定，效率更高（JVM可内联优化）。
- **修饰变量**：
  - 基本类型变量：值不可修改（常量）。
  - 引用类型变量：引用地址不可修改，但对象的内容可以修改（如`final List list = new ArrayList(); list.add(1);`合法）。
  - 类变量（static final）：编译时常量，存储在常量池；实例变量：必须在声明时或构造器中初始化。

---

### 2. static关键字
#### 核心作用
修饰成员变量、方法、代码块、内部类，使其属于**类级别**，而非实例级别。

#### 底层原理与特性
- **修饰成员变量（静态变量）**：
  - 存储在方法区（JDK8+元空间），不属于任何实例，所有实例共享同一份值。
  - 随类加载初始化，仅初始化一次，可通过类名直接访问（`类名.变量名`）。
- **修饰方法（静态方法）**：
  - 属于类，不依赖实例，可直接通过类名调用。
  - 不能访问非静态成员（实例变量/方法），也不能使用`this`/`super`关键字（无实例上下文）。
  - 不能被重写（本质是方法隐藏，子类静态方法仅隐藏父类静态方法，无多态特性）。
- **修饰代码块（静态代码块）**：
  - 类加载时执行一次，用于初始化静态变量，顺序与代码块位置一致。
- **修饰内部类（静态内部类）**：
  - 不依赖外部类实例，只能访问外部类的静态成员，可独立创建实例。

## 四、接口（Interface）与抽象类（Abstract Class）

### 1. 抽象类（Abstract Class）
#### 核心定义
用`abstract`修饰的类，不能被实例化，可包含抽象方法（无方法体）和具体方法，体现“是什么”（is-a）关系。

#### 底层原理与特性
- 抽象类可以有构造器（供子类调用初始化），但不能直接new实例。
- 成员变量：可包含普通成员变量、静态变量、常量。
- 方法：
  - 抽象方法：无方法体，必须被子类实现（除非子类也是抽象类）。
  - 具体方法：可包含方法体，子类可直接继承或重写。
- 单继承：一个类只能继承一个抽象类。

---

### 2. 接口（Interface）
#### 核心定义
用`interface`修饰的抽象契约，定义行为规范，Java 8+支持默认方法和静态方法，体现“能做什么”（has-a/like-a）关系。

#### 底层原理与特性
- 成员变量：默认`public static final`修饰（必须是常量，且必须初始化）。
- 方法：
  - Java 8之前：所有方法默认`public abstract`修饰（抽象方法，无方法体）。
  - Java 8+：新增`default`方法（默认方法，有方法体，实现类可重写）和`static`方法（静态方法，有方法体，属于接口本身）。
  - Java 9+：新增私有方法（辅助默认方法/静态方法，不可被实现类调用）。
- 多实现：一个类可以实现多个接口，解决单继承的局限性。
- 无构造器：接口不能被实例化，也没有构造器。

---

### 3. 接口 vs 抽象类 核心对比
| 对比维度       | 抽象类                     | 接口                       |
|----------------|----------------------------|----------------------------|
| 关系           | is-a（继承关系）           | like-a/has-a（契约关系）   |
| 构造器         | 有构造器                   | 无构造器                   |
| 成员变量       | 可包含普通变量、静态变量   | 仅能包含`public static final`常量 |
| 方法修饰符     | 抽象方法/具体方法，可自定义权限 | 抽象方法默认`public abstract`，Java8+支持default/static |
| 继承/实现限制  | 单继承                     | 多实现                     |
| 设计目的       | 代码复用与模板定义         | 定义行为规范，解耦         |

## 五、高频易错点与底层避坑

1.  **继承的private成员**：子类不能直接访问父类的private成员，只能通过父类的public/protected方法间接访问，即使继承也不包含这些成员。
2.  **方法重写的静态方法**：静态方法不存在重写，子类定义同名静态方法属于“方法隐藏”，调用时以调用者的类型为准（父类调用父类静态方法，子类调用子类静态方法）。
3.  **final修饰引用类型**：final修饰的引用变量，仅保证引用地址不可变，对象内部状态仍可修改（除非对象本身也是不可变类，如String）。
4.  **接口的default方法冲突**：一个类实现多个接口，若多个接口包含同名default方法，实现类必须重写该方法，否则编译报错。
5.  **多态的前提**：只有方法存在重写时才会体现多态，变量不存在多态（父类引用访问的变量以父类定义为准）。

---

## 六、终极背诵总结（面试秒答）
1.  封装：隐藏内部状态，暴露受控接口，用访问修饰符实现权限隔离。
2.  继承：Java单继承，子类复用父类非私有成员，构造器父类先执行，为多态提供基础。
3.  多态：父类引用指向子类对象+方法重写，编译期看引用类型，运行期看对象实际类型，通过方法表实现动态绑定。
4.  重载与重写：重载是同一类中方法名同、参数列表不同，编译期绑定；重写是子类覆盖父类方法，运行期绑定，需遵循方法签名、权限、异常规则。
5.  final：修饰类不可继承、方法不可重写、变量不可修改；static：修饰成员属于类级别，所有实例共享，静态方法不能访问非静态成员。
6.  接口vs抽象类：抽象类体现“是什么”，可包含变量和构造器，单继承；接口体现“能做什么”，仅含常量和抽象方法（Java8+支持default/static），可多实现。
