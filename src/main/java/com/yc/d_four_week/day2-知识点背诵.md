# Day2 对象分配、逃逸分析、栈上分配、TLAB 底层核心知识点背诵笔记 ⭐⭐⭐

---

## 一、对象分配优先级（全路径决策树）

```
new Object()
    │
    ├─→ 逃逸分析：对象是否只在本方法使用？
    │      └─ 是 → 栈上分配（随栈帧销毁自动回收，无 GC 开销）
    │             （还要满足标量替换条件）
    │
    └─→ 逃逸分析未通过 → 尝试 TLAB 分配
           │
           ├─→ TLAB 有空间 → 在 TLAB 内分配（线程私有，无锁）
           │
           └─→ TLAB 不够 → Eden 区分配（可能 CAS 竞争）
                  │
                  ├─→ 大对象（超过 -XX:PretenureSizeThreshold）
                  │      └─→ 直接进入老年代
                  │
                  └─→ 普通对象 → Eden 区
                         │
                         └─→ Minor GC 后 → Survivor → 年龄够了 → 老年代
```

**优先级一句话**：栈上分配 > TLAB > Eden > 老年代（大对象直进）

## 二、逃逸分析（Escape Analysis）⭐⭐

### 2.1 什么是逃逸分析

编译器分析对象动态作用域：如果一个对象**不会**被方法外部引用，就说它**未逃逸**。

```java
// 未逃逸：对象只在方法内部使用
public void test() {
    User user = new User();   // user 只在本方法内使用
    user.setName("张三");     // 不会逃逸 → 可栈上分配
}

// 逃逸：对象被返回或赋值给外部变量
public User test() {
    User user = new User();   // user 被 return 出去了
    return user;              // 逃逸了 → 只能堆分配
}
```

### 2.2 逃逸分析带来的三大优化

| 优化 | 原理 | 效果 |
|---|---|---|
| **栈上分配** | 未逃逸对象分配在栈帧，方法结束自动销毁 | **GC 压力降到零** |
| **锁消除** | 对象只在一个线程使用 → 同步锁无意义 → 去掉 | 消除不必要的同步开销 |
| **标量替换** | 把对象拆成基本类型字段 → 各字段独立分配在栈/寄存器 | 无需创建对象，零 GC |

**标量替换示例**：
```java
// 原始代码
class Point { int x; int y; }
Point p = new Point();  // 如果没逃逸
p.x = 1; p.y = 2;
int sum = p.x + p.y;

// 标量替换后（等价于）
int x = 1;
int y = 2;
int sum = x + y;  // 没有 new Point()，没有对象，没有 GC
```

### 2.3 逃逸分析的局限

- **默认开启**（JDK 6u23+），但依赖 JIT 编译器的分析能力
- 复杂控制流可能导致分析失败 → 回退到堆分配
- **不是所有对象都能栈上分配**：只能放小对象、不能放数组（不确定大小）

## 三、TLAB（Thread Local Allocation Buffer）⭐⭐

### 3.1 核心设计

```
Eden 区
┌──────────────────────────────────────────────┐
│ ┌──────────┐ ┌──────────┐ ┌──────────┐      │
│ │TLAB 线程1 │ │TLAB 线程2 │ │TLAB 线程3 │ ... │
│ │ (1% Eden)│ │ (1% Eden)│ │ (1% Eden)│     │
│ └──────────┘ └──────────┘ └──────────┘      │
│              共享 Eden 区域                    │
└──────────────────────────────────────────────┘
```

**设计目的**：避免多线程在 Eden 区分配对象时的 **CAS 竞争**。

**工作机制**：
1. JVM 给每个线程在 Eden 区分配一小块私有空间（默认占 Eden 的 1%）
2. 线程分配对象时，优先在自己的 TLAB 内用**指针碰撞**（bump-the-pointer）分配 → 无锁，极快
3. TLAB 用完了 → 申请新的 TLAB → 触发 CAS 操作（有竞争但频次低）
4. TLAB 剩余空间放不下当前对象 → 直接在 Eden 共享区分配

**总结**：TLAB 把高频的"无竞争"放在线程内部，低频的"CAS 申请"放在 TLAB 耗尽时 → 极大提升并发分配效率。

## 四、对象进入老年代的 4 种情况（面试必考）⭐⭐⭐

| 序号 | 触发条件 | 机制 | 参数 |
|---|---|---|---|
| **① 大对象** | 对象大小超过阈值 | 直接在老年代分配，避免在 Eden/Survivor 之间来回复制 | `-XX:PretenureSizeThreshold` |
| **② 年龄够了** | Minor GC 熬过次数 ≥ MaxTenuringThreshold | 每活过一次 Minor GC 年龄 +1，默认 15 岁晋升 | `-XX:MaxTenuringThreshold=15` |
| **③ 动态年龄判断** | Survivor 中同龄对象总大小 > Survivor 的 50% | 该年龄及以上的对象直接晋升，不等到 15 | 无需参数 |
| **④ 分配担保** | Minor GC 后 Survivor 放不下存活对象 | 多余对象直接进老年代（老年代要对年轻代做"空间担保"） | — |

### 4.1 动态年龄判断示例

```
Survivor 空间 = 100MB
- 年龄 1 的对象：总计 30MB
- 年龄 2 的对象：总计 25MB  →  年龄 2 的对象总和 > 50MB（50%）
→ 年龄 2 及以上的全部对象直接进老年代！
```

## 五、关键 JVM 参数速查

| 参数 | 含义 |
|---|---|
| `-XX:+DoEscapeAnalysis` | 开启逃逸分析（默认开启） |
| `-XX:+EliminateLocks` | 开启锁消除 |
| `-XX:+EliminateAllocations` | 开启标量替换 |
| `-XX:+UseTLAB` | 开启 TLAB（默认开启） |
| `-XX:TLABSize` | 设置 TLAB 大小 |
| `-XX:PretenureSizeThreshold` | 大对象阈值（仅 Serial/ParNew 有效） |
| `-XX:MaxTenuringThreshold` | 最大晋升年龄（默认 15） |

## 六、高频核心对比

### 栈上分配 vs TLAB

| | 栈上分配 | TLAB |
|---|---|---|
| **位置** | 方法栈帧内 | Eden 区内 |
| **触发条件** | 逃逸分析确定对象未逃逸 | 任何堆分配都优先走 TLAB |
| **生命周期** | 方法结束自动销毁 | 下次 GC 回收 |
| **GC 开销** | **零** | 有（但远小于共享区分配） |
| **限制** | 小对象、非数组、简单控制流 | 对象大小 < TLAB 剩余空间 |

## 七、终极背诵总结

1. **分配优先级**：栈上 > TLAB > Eden > 大对象直入老年代
2. **逃逸分析三优化**：栈上分配 + 锁消除 + 标量替换 → 不逃逸的对象零 GC 开销
3. **TLAB 设计**：Eden 的线程私有小块，用指针碰撞无锁分配，减少 CAS 竞争
4. **进老年代四情况**：大对象直入 / 年龄到 15 / 动态同龄超 50% / Survivor 放不下
5. **标量替换**：对象拆成基本字段，直接用栈/寄存器，连 new 都省了
6. **核心参数**：逃逸分析 JIT 自动做，TLAB 默认开，PretenureSizeThreshold 控制大对象
