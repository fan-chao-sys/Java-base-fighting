## 一、InnoDB vs MyISAM 核心对比

### 1.1 关键差异表

| 对比维度 | InnoDB | MyISAM |
|---|---|---|
| **事务支持** | ✅ 支持 ACID | ❌ 不支持 |
| **锁粒度** | **行级锁**（默认）+ 表锁 | **表级锁** |
| **MVCC** | ✅ 支持（高并发读不阻塞写） | ❌ 不支持 |
| **外键** | ✅ 支持 | ❌ 不支持 |
| **崩溃恢复** | ✅ redo log 保证 | ❌ 崩溃后需修复表 |
| **索引结构** | 聚簇索引（数据即索引） | 非聚簇索引（索引和数据分离） |
| **`COUNT(*)`** | 需扫描索引 | **快**（有行数缓存） |
| **全文索引** | MySQL 5.6+ 支持 | ✅ 原生支持 |
| **存储文件** | `.ibd`（独立表空间）/ `.frm` | `.MYD`(数据) + `.MYI`(索引) + `.frm` |
| **适用场景** | **OLTP**（高并发读写、事务） | 读多写少、数据仓库、日志表 |

### 1.2 选型原则

```
需要事务 / 行锁 / 崩溃恢复 / 外键 → InnoDB（MySQL 5.5+ 默认，绝对首选）
只读或极少写 / 不需要事务 / 报表统计 → MyISAM（已边缘化，不推荐新项目使用）
```

## 二、InnoDB 页结构（Page）

### 2.1 核心定义

**页（Page）是 InnoDB 磁盘管理的最小单位，默认 16KB。**

每次从磁盘读写数据，最少读/写一个完整的页。所有数据行都存储在页中。

### 2.2 页的 7 部分结构

```
┌─────────────────────────────┐
│      File Header（38字节）    │ ← 页编号、上一页/下一页指针、页类型、LSN
├─────────────────────────────┤
│     Page Header（56字节）     │ ← 槽数量、记录数、空闲空间指针、最后插入位置
├─────────────────────────────┤
│  Infimum + Supremum Records  │ ← 最小记录和最大记录（两条虚拟记录，边界哨兵）
├─────────────────────────────┤
│      User Records            │ ← 实际存放的用户数据行（按主键顺序链表排列）
├─────────────────────────────┤
│      Free Space              │ ← 页中未被使用的空间（新记录从这里分配）
├─────────────────────────────┤
│     Page Directory           │ ← 槽（Slot）数组，分组二分查找定位记录
├─────────────────────────────┤
│     File Trailer（8字节）     │ ← 校验和 + LSN，保证页完整性写入
└─────────────────────────────┘
```

### 2.3 Page Directory 如何加速页内查找

页内记录按主键顺序组成**单向链表**，但每条记录都遍历太慢：
1. 把记录分成若干组，每组 4~8 条
2. 每组最后一条记录的页内偏移量存入 **Page Directory（槽数组）**
3. 查找时 → 二分查找槽数组定位组 → 在组内顺序遍历 → 时间复杂度 O(log n)

## 三、行格式（Row Format）

### 3.1 四种行格式对比

| 行格式 | 特点 | 溢出页处理 |
|---|---|---|
| **Compact** | 紧凑存储，变长字段长度列表 + NULL 位图 | 768 字节前缀 + 20 字节溢出页指针 |
| **Dynamic**（默认） | 与 Compact 类似 | 完全溢出：只存 20 字节溢出页指针 |
| **Compressed** | 同 Dynamic + 支持页级压缩 | 同 Dynamic |
| **Redundant** | 旧格式（MySQL 5.0 前），已弃用 | 同 Compact |

### 3.2 Dynamic 行格式结构

```
┌────────────────┬──────────┬───────────────┬─────┬─────┬─────┐
│ 变长字段长度列表  │ NULL位图  │ 记录头信息(5字节) │ col1 │ col2 │ col3 │
│ (逆序，1~2字节/列)│ (1字节/8列)│                │      │      │      │
└────────────────┴──────────┴───────────────┴─────┴─────┴─────┘
```

**溢出页机制**（DYNAMIC）：
- 当变长列（VARCHAR/TEXT/BLOB）超过一行放不下时
- 整列数据移到**溢出页**（Overflow Page）
- 原位置只存 20 字节溢出页指针

## 四、InnoDB 表空间

| 表空间类型 | 作用 | 参数 |
|---|---|---|
| **系统表空间** | 存储数据字典、undo log、doublewrite buffer 等 | `ibdata1` |
| **独立表空间** | 每个表一个 `.ibd` 文件 | `innodb_file_per_table=ON`（默认） |
| **undo 表空间** | 存储 undo log（回滚日志） | `innodb_undo_tablespaces` |
| **临时表空间** | 存储临时表、排序中间结果 | `ibtmp1` |

## 五、InnoDB 内存结构

### 5.1 四大核心组件

```
┌─────────────────────────────────────────────┐
│              InnoDB 内存（Buffer Pool）       │
├─────────────────────────────────────────────┤
│  Buffer Pool         │  缓存数据页和索引页     │ ← 占内存 70%~80%
│  （缓冲池）           │  LRU 淘汰策略          │
├──────────────────────┼───────────────────────┤
│  Change Buffer       │  缓存非唯一二级索引的    │ ← DML 操作先写这里
│  （写缓冲）           │  修改，减少随机 IO       │
├──────────────────────┼───────────────────────┤
│  Adaptive Hash Index │  对热点数据页自动构建     │ ← B+树查找优化为 O(1)
│  （自适应哈希索引）    │  哈希索引，等价查询加速    │
├──────────────────────┼───────────────────────┤
│  Log Buffer          │  redo log 缓冲区       │ ← 默认 16MB，写满刷盘
│  （日志缓冲）         │  WAL 策略先写日志        │
└──────────────────────┴───────────────────────┘
```

### 5.2 Buffer Pool LRU 链表

```
Buffer Pool 内部用变种 LRU 链表管理页的淘汰：

  Young区（热数据，63%）    Old区（冷数据，37%）
  ┌──────────────────┬──────────────────┐
  │   频繁访问的页     │   新读入的页       │
  │   不易被淘汰       │   首次访问放中点    │ → 防止全表扫描冲掉热数据
  └──────────────────┴──────────────────┘
        Midpoint（5/8 位置）
```

**关键设计**：全表扫描的页放入 Old 区 → 再次被访问才晋升到 Young 区 → 避免一次性扫描把热数据挤出。

## 六、终极背诵总结

1. **InnoDB vs MyISAM**：InnoDB 有事务/行锁/MVCC/崩溃恢复；MyISAM 表锁/无事务/只适合读多写少
2. **页 7 部分**：File Header → Page Header → Infimum+Supremum → User Records → Free Space → Page Directory → File Trailer
3. **Page Directory**：槽数组 + 二分查找，页内 O(log n) 定位
4. **行格式选 Dynamic**：变长字段完全溢出，只存 20 字节指针
5. **内存四大件**：Buffer Pool（缓存数据页）+ Change Buffer（缓冲二级索引写）+ AHI（热点哈希索引）+ Log Buffer（redo 缓冲）
6. **LRU 分 Young/Old**：Old 区做全表扫描缓冲，防止冲掉热数据
