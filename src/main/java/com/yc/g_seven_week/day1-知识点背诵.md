## 一、Redis 为什么快

| 维度 | 原因 |
|---|---|
| **纯内存操作** | 数据在内存中，读写纳秒级 |
| **单线程模型** | 无多线程竞争、无上下文切换开销（6.0+ 网络 IO 多线程，执行仍是单线程） |
| **IO 多路复用** | epoll 机制，一个线程处理多个连接 |
| **高效数据结构** | SDS / 跳跃表 / 压缩列表 都是为 Redis 定制的 |
| **RESP 协议简单** | 文本协议，解析快 |

## 二、5 大数据类型及场景

| 类型 | 底层结构（小数据→大数据） | 典型场景 |
|---|---|---|
| **String** | int/embstr → raw（SDS） | 缓存、计数、分布式锁、Session |
| **Hash** | ziplist → hashtable | 对象存储（用户信息）、购物车 |
| **List** | quicklist（ziplist+linkedlist） | 消息队列、最新列表、时间线 |
| **Set** | intset → hashtable | 去重、抽奖、共同好友、标签 |
| **ZSet** | ziplist → skiplist+dict | 排行榜、延时队列、带权重的集合 |

### 2.1 核心底层结构

| 结构 | 特点 |
|---|---|
| **SDS（简单动态字符串）** | 二进制安全、O(1) 获取长度、预分配空间防频繁分配 |
| **ziplist（压缩列表）** | 连续内存块，省内存但增删 O(n)，小数据用 |
| **quicklist** | ziplist 的链表，List 的默认结构 |
| **skiplist（跳跃表）** | 多层索引链表，O(log n) 查找，ZSet 核心 |
| **intset（整数集合）** | 纯整数 Set 小数据专用 |

## 三、持久化

### 3.1 RDB vs AOF

| | RDB（快照） | AOF（追加日志） |
|---|---|---|
| **原理** | `bgsave` fork 子进程全量快照 | 每次写操作追加到日志 |
| **优点** | 恢复快、文件小、对性能影响小 | 数据安全高、可读性好 |
| **缺点** | 可能丢最近几分钟数据 | 文件大、恢复慢 |
| **适用** | 备份/灾备 | 数据安全要求高 |

**混合持久化（Redis 4.0+）**：RDB 快照 + 快照后的 AOF 增量 → 既快又全。

### 3.2 AOF 刷盘策略

| 值 | 策略 | 持久性 |
|---|---|---|
| `always` | 每次写后 fsync | 最安全，性能最差 |
| `everysec`（默认） | 每秒 fsync 一次 | 最多丢 1 秒数据，推荐 |
| `no` | 由 OS 决定 | 可能丢大量数据 |

## 四、过期与淘汰策略

### 4.1 过期删除策略

- **惰性删除**：访问 key 时检查是否过期 → 过期则删除
- **定期删除**：每 100ms 随机抽样 20 个 key，删除其中过期的

**两者结合**，平衡 CPU 和内存。

### 4.2 内存淘汰策略（8 种）

| 策略 | 范围 | 行为 |
|---|---|---|
| `noeviction` | — | 不淘汰，写报错 |
| `allkeys-lru` | 所有 key | 淘汰最近最少使用的 |
| `volatile-lru` | 有过期时间的 key | 同上 |
| `allkeys-lfu` | 所有 key | 淘汰使用频次最低的 |
| `volatile-lfu` | 有过期时间的 key | 同上 |
| `allkeys-random` | 所有 key | 随机淘汰 |
| `volatile-random` | 有过期时间的 key | 同上 |
| `volatile-ttl` | 有过期时间的 key | 淘汰最快要过期的 |

## 五、终极背诵总结

1. **为什么快**：内存操作 + 单线程 + IO 多路复用 + 高效数据结构 + RESP 协议
2. **5 大类型**：String→缓存计数、Hash→对象、List→队列、Set→去重、ZSet→排行榜
3. **底层结构变化**：小数据用紧凑结构（ziplist/intset），大数据转标准结构（skiplist/hashtable）
4. **RDB**：全量快照，恢复快但可能丢数据；**AOF**：追加日志，安全但恢复慢
5. **过期**：惰性删除 + 定期删除；**淘汰**：LRU/LFU/TTL 等 8 种
