## 一、TCP vs UDP

| | TCP | UDP |
|---|---|---|
| **连接** | 面向连接 | 无连接 |
| **可靠性** | 可靠（确认+重传+序号） | 不可靠 |
| **有序性** | 有序 | 无序 |
| **流量控制** | 滑动窗口 | 无 |
| **拥塞控制** | 慢启动/拥塞避免/快重传 | 无 |
| **速度** | 慢 | **快** |
| **应用** | HTTP/FTP/邮件 | 视频/DNS/直播/游戏 |

## 二、三次握手

```
Client                              Server
  │──────── SYN seq=x ──────────────→│  ① 客户端：我要连你
  │←──── SYN+ACK seq=y ack=x+1 ────│  ② 服务端：我收到，确认你能收到吗
  │──────── ACK ack=y+1 ───────────→│  ③ 客户端：收到确认，连接建立
  │                                   │
  │        连接建立，开始传数据           │
```

### 2.1 为什么是三次不是两次/四次？

- **两次不行**：Client 发 SYN → Server 回 SYN+ACK，Server 认为连接建立并开始等数据。但如果 Client 的 SYN 是**旧连接超时重发的**，Client 已经不要了，Server 傻等 → 浪费资源
- **三次刚好**：Client 必须再回 ACK，Server 收到后才建立连接。老的无效 SYN 不会触发第三次 ACK → Server 不会浪费资源
- **四次太多**：没必要，三次就能确认双方收发能力正常

## 三、四次挥手

```
Client                              Server
  │──────── FIN seq=u ──────────────→│  ① 客户端：我不发数据了
  │←──────── ACK ack=u+1 ──────────│  ② 服务端：知道了（但可能还有数据要发）
  │                                   │
  │  ← 服务端可能还在发剩余数据 →        │
  │                                   │
  │←──────── FIN seq=v ──────────────│  ③ 服务端：我的数据也发完了
  │──────── ACK ack=v+1 ───────────→│  ④ 客户端：收到，等 2MSL 后关闭
  │ 等待 2MSL（最长报文寿命×2）           │
  │ 关闭                               │
```

### 3.1 为什么是四次挥手？

TCP 是**全双工**的：`FIN` 只表示"我不发数据了"但还能**接收数据**。服务端可能还有数据要发 → ACK 和 FIN 必须分开发 → 四次。

### 3.2 为什么最后要等 2MSL？

1. 确保最后一个 ACK 能到达服务端（如果丢了，服务端会重发 FIN）
2. 让网络中残留的旧连接报文消失，不影响新连接

## 四、HTTP vs HTTPS

| | HTTP | HTTPS |
|---|---|---|
| **安全性** | 明文传输 | TLS/SSL 加密 |
| **端口** | 80 | 443 |
| **加密层** | 无 | TLS 1.2/1.3 |
| **证书** | 不需要 | CA 签发证书 |
| **握手** | TCP 三次握手 | TCP + **TLS 握手**（多 2 RTT） |

### 4.1 HTTPS = HTTP + TLS 三层保障

1. **加密传输**：对称 + 非对称混合加密（TLS 握手用非对称交换对称密钥，之后用对称加密传数据）
2. **身份认证**：CA 数字证书（证明你是你就是你）
3. **数据完整性**：MAC 消息认证码（防中间人篡改）

## 六、终极背诵总结

1. **TCP vs UDP**：TCP 可靠有连接，UDP 快无连接
2. **三次握手**：SYN → SYN+ACK → ACK（防旧连接请求）
3. **四次挥手**：FIN → ACK → FIN → ACK（全双工，ACK 和 FIN 分开）
4. **等 2MSL**：确保 ACK 到达 + 清空网络残留包
5. **HTTPS = HTTP + TLS**：加密传输 + CA 身份认证 + MAC 防篡改
